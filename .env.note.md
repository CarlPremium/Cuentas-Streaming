# Environment Variables Configuration

## Important: This Project Uses `.env`

This project is configured to use `.env` (not `.env.local`) for environment variables.

### Why `.env` instead of `.env.local`?

- Simpler configuration
- Single source of truth
- Already in use by the project

### Security

âœ… **`.env` is protected in `.gitignore`**

Your `.env` file will NOT be committed to Git. It's safe to store your secrets there.

### Setup

1. Copy the example file:
   ```bash
   copy .env.example .env
   ```

2. Edit `.env` with your actual values

3. Never commit `.env` to version control

### Next.js Environment Variable Loading Order

Next.js loads environment variables in this order (later files override earlier ones):

1. `.env` - All environments
2. `.env.local` - All environments (ignored by Git by default)
3. `.env.development` - Development only
4. `.env.development.local` - Development only (ignored by Git)
5. `.env.production` - Production only
6. `.env.production.local` - Production only (ignored by Git)

**This project uses:** `.env` for all environments

### Required Variables

```env
# Application
NEXT_PUBLIC_APP_NAME=
NEXT_PUBLIC_APP_URL=

# Secret Key
SECRET_KEY=

# Supabase
NEXT_PUBLIC_SUPABASE_PROJECT_ID=
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
NEXT_PUBLIC_SUPABASE_STORAGE_BUCKET=
SUPABASE_SERVICE_ROLE_KEY=
```

### Troubleshooting

**Variables not loading?**
- Restart your dev server after changing `.env`
- Check file is named exactly `.env` (not `.env.txt`)
- Verify file is in project root directory
- Check for typos in variable names

**Want to use different files for dev/prod?**
- Create `.env.development` for dev-specific vars
- Create `.env.production` for prod-specific vars
- Keep common vars in `.env`
